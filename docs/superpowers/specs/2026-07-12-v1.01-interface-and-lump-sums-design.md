# Retirement Income Simulator v1.01 Interface and Lump Sums Design

## Goal

Deliver a focused v1.01 release that adds auditable one-off lump-sum withdrawals and improves desktop layout, section navigation, chart readability, and survival context while preserving the dependency-free single-HTML-file architecture.

The application should remain below a practical 250-300 KB ceiling. No build tooling, runtime dependencies, or module split is introduced.

## Release scope

The deterministic simulator advances from v1.00 to v1.01. Existing v1.00 JSON remains importable through schema migration. The Monte Carlo v0.5 engine remains unchanged apart from documentation of its existing conditional import limitation; a GitHub issue will track support for populated Other income/assets and active Defined Benefit/UK Pension income.

## Lump-sum withdrawal model

Add a repeatable `lumpSumWithdrawals` collection to the scenario. Each entry contains:

- amount;
- reason;
- year; and
- requested funding source.

The default funding source is `Automatic - use household drawdown order`. Explicit choices include Cash, Savings, either person's super, and eligible individual shareholdings or other assets.

An explicit source is used first. If it cannot cover the amount, the remainder follows the household drawdown order. A lump sum reduces assets in its selected year but is excluded from annual income and does not satisfy the requested-income target.

Projection output retains the total lump sum and actual source breakdown. Multiple withdrawals may occur in one year.

## Event audit trail

The projection Event column is the common audit trail for dated actions:

- lump-sum withdrawals include reason, amount, and actual funding sources;
- existing super transition events remain;
- shareholding and other asset sales include the asset name and proceeds; and
- multiple events in one year are listed clearly in the same cell.

Chart event markers and tooltips consume the same event data so event descriptions are not reimplemented separately.

## Interface and layout

### Header

Use a compact desktop header row with the title and v1.01 label on the left and New, Reset, Import JSON, Export JSON, and Theme controls aligned on the right. Add subtle visual separation between identity and actions. Retain one short privacy/context subtitle below the row. On narrow screens, actions wrap beneath the title.

### Left panel

The desktop left panel defaults to 24% of viewport width, remains draggable, and is constrained between 320 CSS pixels and 40% of viewport width.

Every section uses the same full-width chevron header and can independently expand or collapse. Initial state:

- Scenario: expanded;
- Returns & Inflation: expanded;
- People: expanded;
- Cash & Savings: collapsed;
- Lump Sum Withdrawals: collapsed;
- Shareholdings: collapsed;
- Defined Benefit/UK Pensions: collapsed;
- Other Income & Assets: collapsed; and
- Household: collapsed.

Rename `Cash and savings` to `Cash & Savings`. Place Lump Sum Withdrawals immediately below it. Defined Benefit/UK Pensions defaults to the Australian Defined Benefit path rather than UK State Pension.

### Chart and table split

Give the chart approximately 20% more vertical space by default, taking that space from the visible table viewport while keeping the overall dashboard height broadly stable. Add a horizontal splitter between chart and table that supports pointer dragging, keyboard adjustment, and double-click reset. Persist the user's split locally.

Responsive layouts preserve the current mobile stacking behaviour.

## Charts

### Theme-aware palette

Keep source identity consistent between themes while defining theme-specific values: darker, more saturated chart colours for light mode and lighter equivalents for dark mode. Grid lines, labels, bars, lines, legends, focus states, and tooltips must retain usable contrast in both themes.

### Lump-sum markers

Display each lump sum as a separate labelled block approximately 10 CSS pixels above that year's income stack. It is visually distinct from income and never included in the stack total. The chart scale accounts for the block so it is not clipped.

The tooltip includes amount, reason, requested source, and actual funding sources.

### Survival ribbon

Replace the current graduated fade with a compact ribbon at the bottom of both Income and Balance charts. The ribbon remains visually separate from financial data.

Show one labelled lane per active person. Each lane uses the person's name and marks the ages corresponding to illustrative 80%, 50%, 20%, and 5% survival probabilities. A single-person scenario shows one lane; a couple shows two. Both charts share one survival calculation and ribbon renderer.

## Validation and compatibility

- Migrate v1.00 scenarios by adding an empty lump-sum collection.
- Require an amount greater than zero, a non-empty trimmed reason, a year within the model horizon, and a valid source.
- Use fallback drawdown when an explicit source is unavailable or depleted, and report actual sources in events and tooltips.
- Preserve local autosave and JSON import/export round-tripping.
- Preserve Monte Carlo v0.5's current clear, conditional rejection of unsupported imported cash flows.

## Verification

Extend the deterministic regression suite to cover:

- v1.00-to-v1.01 migration;
- lump-sum accounting and exclusion from income;
- explicit source priority and fallback;
- multiple withdrawals and multiple Event entries in one year;
- asset-sale Event entries;
- JSON and local-state round-trips;
- Australian Defined Benefit default;
- labels and initial accordion state;
- splitter persistence/reset hooks;
- theme-aware chart tokens;
- lump-sum chart hooks; and
- shared survival-ribbon hooks.

Run both deterministic and Monte Carlo suites, verify file size remains under the agreed ceiling, run `git diff --check`, and visually inspect light and dark themes at desktop and narrow widths in a real browser.

## Out of scope

- Reproducing deterministic Other income/assets or Defined Benefit/UK Pension cash flows in Monte Carlo v0.5.
- Splitting the simulator into multiple runtime files.
- Adding dependencies or a build pipeline.
- Changing the underlying tax, Age Pension, investment-return, or drawdown policy beyond the funding needed for lump sums.

## Approved refinement pass

The following refinements were approved after browser review:

- Default the fictional sample to the Treasury inflation schedule. Keep the live real-return readout and explain in the interface that real return means nominal return minus inflation, so changing the inflation assumption changes the displayed spread.
- Move the local-storage and optional-network-request sentence to the right beneath the header action buttons on desktop; allow it to wrap naturally on narrow screens.
- Use the same blue accent for every left-panel section heading. Reserve multi-colour semantics for chart series rather than implying a one-to-one section/chart mapping.
- Replace the two-person survival lanes with one Person 1 ribbon on both charts. Match the reference treatment as closely as the existing Canvas architecture permits: a single translucent band labelled `Chance of living to each age`, with 80%, 50%, 20%, and 5% markers when those thresholds fall inside the model horizon.
- Label the horizontal age axis at conventional ages divisible by five, plus the final age only when needed for orientation.
- Add two fictional lump sums to the sample using relative years so the scenario remains current: `New car`, AUD 20,000, start year + 3; and `European holiday`, AUD 50,000, start year + 5. Both default to automatic household drawdown order.
- Retain the README trust statement, current interface screenshot, SuperCalcs inspiration acknowledgement, and Event-column audit trail already delivered in the first pass.
