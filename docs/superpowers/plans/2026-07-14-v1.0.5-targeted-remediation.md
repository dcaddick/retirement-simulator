```js
function projectionNetTax({
  taxableNominal,
  rebateNominal,
  inflationFactor,
  year,
  seniorEligible
}) {
  const factor = inflationFactor > 0 ? inflationFactor : 1;
  const taxableReal = taxableNominal / factor;
  const rebateReal = rebateNominal / factor;
  const grossReal = incomeTaxResident(taxableNominal, year) / factor +
    medicareLevy(taxableReal, seniorEligible);
  const offsetsReal = litoOffset(taxableNominal) / factor +
    (seniorEligible ? saptoOffset(rebateReal) : 0);
  return Math.max(0, grossReal - offsetsReal);
}
```

```js
function csvCell(value) {
  const text = String(value ?? '');
  const safe = /^[=+\-@]/.test(text) ? `'${text}` : text;
  return `"${safe.replaceAll('"', '""')}"`;
}

function csvFromMatrix(matrix) {
  return '\uFEFF' + matrix
    .map(row => row.map(csvCell).join(','))
    .join('\r\n') + '\r\n';
}
```

- [ ] **Step 4: Add DOM-to-matrix and download orchestration in the UI script**

Add:

```js
function visibleTableMatrix(table) {
  return [...table.rows].map(row =>
    [...row.cells].map(cell => cell.textContent.trim())
  );
}

function projectionCsvFilename(scenario) {
  const basis = displayMode === 'nominal' ? 'future-dollars' : 'todays-dollars';
  const date = new Date().toISOString().slice(0, 10);
  return `${slugify(scenario.scenarioName || 'retirement-scenario')}-projection-${basis}-${date}.csv`;
}

function downloadProjectionCsv() {
  const table = $('#tbl');
  if (!currentProjection || !table?.tHead || !table.tBodies[0]?.rows.length) {
    throw new Error('No valid projection table is available.');
  }
  const blob = new Blob(
    [core.csvFromMatrix(visibleTableMatrix(table))],
    { type: 'text/csv;charset=utf-8' }
  );
  const url = URL.createObjectURL(blob);
  const anchor = document.createElement('a');
  anchor.href = url;
  anchor.download = projectionCsvFilename(currentScenario);
  anchor.click();
  URL.revokeObjectURL(url);
}
```

```markdown
# Deferred Review Register

This register tracks known modelling limitations and review questions that are not resolved in v1.0.5. It is not professional tax, financial, accounting, actuarial, or legal advice.

## Status definitions

- **Deferred:** acknowledged and intentionally outside the current release.
- **In review:** an owner or qualified reviewer is assessing the item.
- **Planned:** a separately approved design exists.
- **Resolved:** a named release or recorded decision closed the item.

## Register

| ID | Status | Current behaviour | Likely impact | Reason deferred | Required review or decision | Next action |
|---|---|---|---|---|---|---|
| AIPR-003-SHARES-STATIC | Deferred | Shareholdings use a manually entered static price and produce no dividends or franking credits. | Equity-heavy scenarios can understate assets and income over long horizons. | Requires return, income, tax, cost-base and UI assumptions. | Accounting, tax and product scope. | Design a separately testable share total-return model. |
| AIPR-003-AP-AGEGAP | Deferred | Age Pension remains zero until both partners reach pension age. | Can understate income for age-gapped couples. | One-partner eligibility and couple-rate mechanics need professional confirmation. | Financial-advice and social-security review. | Specify and test one-eligible-partner calculations. |
| AIPR-003-MORTALITY | Deferred | Survival probabilities are illustrative; both people remain in every cash-flow year. | Omits first-death spending and survivor-rate transitions. | Requires a first-death event model and survivor assumptions. | Actuarial and financial-planning scope. | Design an explicit survivor-state model or approve limitation-only treatment. |
```
