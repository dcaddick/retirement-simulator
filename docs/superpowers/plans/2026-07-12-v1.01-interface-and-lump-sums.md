- [ ] **Step 3: Implement reusable source-first funding**

Keep normal income drawdown unchanged. Add a helper that prepends an explicit source to a de-duplicated fallback order:

```js
function drawForLumpSum(state, tiers, amount, requestedSource) {
  const liquidSource = ['cash', 'savings', 'p0Super', 'p1Super'].includes(requestedSource)
    ? requestedSource : null;
  const automatic = requestedSource === 'automatic';
  const first = liquidSource ? [[{ source: liquidSource, weight: 1 }]] : [];
  const fallback = tiers
    .map(tier => tier.filter(entry => automatic || entry.source !== liquidSource))
    .filter(tier => tier.length);
  return drawFromTiers(state, [...first, ...fallback], amount);
}
```

Use stable dynamic source keys `share:<holding.id>` and `asset:<asset.id>` for named holdings. Before `drawForLumpSum`, liquidate up to the requested amount from that named asset, deposit the proceeds into savings, and then withdraw those proceeds. For a shareholding, reduce units proportionately, calculate the realised gain using the same cost-base and CGT-discount rules as `applyShareSales`, and return the discounted gain to that year's tax calculation. For another asset, reduce `asset.value` by the proceeds. Mark an asset sold only when its residual units/value reaches zero. Emit a sale/disposal Event and let the normal household tiers fund any remainder.

```js
function liquidateNamedSource(state, scenario, source, amount, inflationFactor) {
  if (source.startsWith('share:')) {
    const holding = state.holdings.find(item => item.id === source.slice(6));
    if (!holding || holding.sold) return { proceeds: 0, events: [], discountedGainByPerson: [0, 0] };
    const marketValue = shareValueAud(holding);
    const proceeds = Math.min(amount, marketValue);
    const fraction = marketValue > 0 ? proceeds / marketValue : 0;
    const costBaseUsed = holding.costBaseAud * fraction;
    const gain = Math.max(0, proceeds - costBaseUsed);
    holding.units *= 1 - fraction;
    holding.costBaseAud -= costBaseUsed;
    holding.sold = holding.units <= 1e-8;
    state.savings += proceeds;
    return namedSaleResult(holding, proceeds, gain, inflationFactor, scenario.people);
  }
  if (source.startsWith('asset:')) {
    const asset = state.otherAssets.find(item => item.id === source.slice(6));
    const proceeds = Math.min(amount, asset?.sold ? 0 : asset?.value ?? 0);
    if (asset) {
      asset.value -= proceeds;
      asset.sold = asset.value <= 1e-8;
      state.savings += proceeds;
    }
    return namedAssetResult(asset, proceeds, inflationFactor);
  }
  return { proceeds: 0, events: [], discountedGainByPerson: [0, 0] };
}
```

- [ ] **Step 4: Apply withdrawals after scheduled sales and before year-end growth**

For entries matching the projection year, call `drawForLumpSum`, aggregate actual draws, and append one event per entry:

```js

  const liquidation = liquidateNamedSource(

  events.push(...liquidation.events);

  Object.keys(lumpSumDraws).forEach(source => {
```

- [ ] **Step 3: Restore and export the illustrative Gompertz calculation**

Use the original simulator's documented illustrative curve:

```js
function survivalProbability(currentAge, age) {
  const A = 5.55e-6;
  const G = 0.11;
  const cumulativeHazard = value => (A / G) * Math.exp(G * value);
  return age <= currentAge
    ? 1
    : Math.exp(-(cumulativeHazard(age) - cumulativeHazard(currentAge)));
}

function survivalMilestones(currentAge, maxAge) {
  return Object.fromEntries([0.8, 0.5, 0.2, 0.05].map(probability => {
    for (let age = currentAge; age <= maxAge; age += 1) {
      if (survivalProbability(currentAge, age) <= probability) return [probability, age];
    }
    return [probability, maxAge];
  }));
}
```

Export both helpers on `RetirementSimulatorCore` and document that the curve is illustrative, not actuarial advice.
