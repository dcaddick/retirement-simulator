# v1.0.6 Tax Ownership, CGT and Share Returns Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Release deterministic simulator v1.0.6 with owned other income, per-person capital-loss carry-forward, reconciled CGT funding, and a simplified share total-return model covering price growth, dividends, and Australian franking credits.

**Architecture:** Keep the dependency-free single-file application and add a bounded per-person tax ledger inside its existing core/UI separation. Schema 11 supplies inert defaults for all new persisted fields; share sales emit undiscounted records, the ledger owns loss/discount/franking ordering, and a separate expense pass funds CGT without treating proceeds as retirement income. Monte Carlo v0.5 accepts schema 11 only when the new deterministic share-return fields are inert.

**Tech Stack:** Self-contained HTML/CSS/JavaScript, Node.js core/contract tests, Playwright CLI browser verification, Git/GitHub CLI, official ATO guidance linked from the approved specification.

- [ ] **Step 3: Implement annual growth and month exposure**

Add beside `shareValueAud`:

```js
function beginShareYear(state) {
  for (const holding of state.holdings) {
    holding.dividendQuantityMonths = 0;
    holding.dividendCursorMonth = 1;
    if (holding.sold) continue;
    const growthFactor = Math.max(0, 1 + holding.priceGrowthPct / 100);
    holding.price *= growthFactor;
  }
}

function recordDividendExposure(holding, throughMonth) {
  const month = Math.max(1, Math.min(12, Math.trunc(throughMonth)));
  const cursor = holding.dividendCursorMonth ?? 1;
  if (month < cursor) return;
  holding.dividendQuantityMonths =
    (holding.dividendQuantityMonths ?? 0) +
    holding.quantity * (month - cursor + 1);
  holding.dividendCursorMonth = month + 1;
}

function finishDividendExposure(holding) {
  recordDividendExposure(holding, 12);
  return holding.dividendQuantityMonths ?? 0;
}
```

Call `beginShareYear(state)` inside `projectScenario` immediately after capturing `openingSavings` and before scheduled sales. Zero-growth defaults preserve the v1.0.5 baseline.

function capitalLedgerForPeople(saleRecords, openingLosses = [0, 0]) {
  const people = [0, 1].map(index => ({
    openingLoss: Math.max(0, openingLosses[index] ?? 0),
    currentLoss: 0,
    nonDiscountGain: 0,
    discountGain: 0,
    lossAppliedNonDiscount: 0,
    lossAppliedDiscount: 0,
    netCapitalGainNominal: 0,
    closingLoss: 0
  }));

  for (const record of saleRecords) {
    const allocations = allocateOwnedAmount(
      Math.abs(record.capitalResultNominal), record.owner);
    allocations.forEach((amount, index) => {
      if (record.capitalResultNominal < 0) people[index].currentLoss += amount;
      else if (record.discountEligible) people[index].discountGain += amount;
      else people[index].nonDiscountGain += amount;
    });
  }

  for (const person of people) {
    let availableLoss = person.openingLoss + person.currentLoss;
    person.lossAppliedNonDiscount = Math.min(person.nonDiscountGain, availableLoss);
    availableLoss -= person.lossAppliedNonDiscount;
    person.lossAppliedDiscount = Math.min(person.discountGain, availableLoss);
    availableLoss -= person.lossAppliedDiscount;
    const remainingNonDiscount =
      person.nonDiscountGain - person.lossAppliedNonDiscount;
    const remainingDiscount = person.discountGain - person.lossAppliedDiscount;
    person.netCapitalGainNominal = remainingNonDiscount + remainingDiscount * 0.5;
    person.closingLoss = availableLoss;
  }

  return { people, closingLosses: people.map(person => person.closingLoss) };
}
