# Retirement Income Simulator v1.03 Collapsible Lump Sums Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship v1.03 with completed lump sums collapsed into aligned `›  $100K  Holiday  Jun 2032  ☑` modelling rows that expand for editing and removal.

**Architecture:** Preserve the dependency-free single-file application. Advance deterministic scenarios to schema 9 for an intended month, keep annual calculations unchanged, and keep expansion state in an in-memory `Set` keyed by stable lump-sum IDs. Extend the existing Node structural/core tests and Monte Carlo adapter rather than adding runtime dependencies.

**Tech Stack:** HTML5, CSS Grid, vanilla JavaScript, Node.js `vm` regression tests, Playwright browser verification already present in the repository.

Add UI-only helpers near `renderLumpSums`:

```js
const expandedLumpIds = new Set();
const MONTH_NAMES = ['Jan','Feb','Mar','Apr','May','Jun','Jul','Aug','Sep','Oct','Nov','Dec'];

function compactMoney(value) {
  const amount = Number(value) || 0;
  if (Math.abs(amount) >= 1_000_000) return `$${(amount / 1_000_000).toFixed(amount % 1_000_000 ? 1 : 0)}M`;
  if (Math.abs(amount) >= 1_000) return `$${(amount / 1_000).toFixed(amount % 1_000 ? 1 : 0)}K`;
  return formatMoney(amount);
}

function lumpDateLabel(item) {
  return `${MONTH_NAMES[item.month - 1] ?? 'Jan'} ${item.year}`;
}

function lumpIsComplete(item) {
  return Number(item.amount) > 0 && String(item.reason ?? '').trim() &&
    Number.isInteger(item.year) && Number.isInteger(item.month) && item.month >= 1 && item.month <= 12 && item.source;
}
```
